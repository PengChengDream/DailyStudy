# 主持人打开一扇空门后，为什么换门不是 50 对 50？

日期：2026-05-29


## 1. 智商题：三门问题

- 题目：三扇门后有一辆车和两只羊。你先选一扇门，主持人知道答案并打开另一扇有羊的门。此时换门是否更优？
- 考察点：条件概率、信息泄露、策略分析

<details>
<summary>参考答案</summary>

应该换门，中奖概率从 1/3 提升到 2/3。初始选中车的概率是 1/3，换门会输；初始选中羊的概率是 2/3，主持人会打开另一只羊，换门会赢。主持人的动作把初始未选的两扇门的 2/3 概率集中到剩下那扇未开的门上。

</details>

## 2. 经典 ML/DL：类别极不均衡时为什么 AUC 可能比 Accuracy 更有用？

- 题目：在正负样本极不均衡的二分类任务里，Accuracy、Precision/Recall、AUC 分别容易踩什么坑？
- 考察点：分类评估、排序指标、数据不均衡

<details>
<summary>参考答案</summary>

Accuracy 在类别不均衡时可能被多数类主导，例如全预测负类也很高。Precision/Recall 更关注正类，但依赖具体阈值。AUC 衡量随机正样本排在随机负样本前面的概率，能评价整体排序质量，不依赖单一阈值。不过当业务只关心极低 FPR 区间或 top-k 召回时，全局 AUC 也可能过于乐观，需要补充 PR-AUC、Recall@K 或局部 AUC。

</details>

## 3. 热门技术：Agent 的 memory 应该存什么，不应该存什么？

- 题目：设计一个面向工作流的 Agent，长期记忆、短期上下文和外部知识库分别应该承担什么职责？
- 考察点：Agent、Memory、隐私

<details>
<summary>参考答案</summary>

短期上下文保存当前任务目标、最近对话和工具返回；长期记忆保存用户稳定偏好、项目约束和可复用经验；外部知识库保存可检索事实和文档。不要把敏感信息、临时错误状态或未经验证的模型猜测直接写入长期记忆。工程上需要记忆写入门控、过期机制、来源追踪、用户可编辑和权限隔离。

</details>

## 4. 热门技术：多模态模型为什么需要视觉 token 和文本 token 对齐？

- 题目：视觉编码器接到 LLM 前，常见的 projector、Q-Former 或 cross-attention 分别在解决什么问题？
- 考察点：多模态、表示对齐、视觉语言模型

<details>
<summary>参考答案</summary>

视觉编码器输出的是图像 patch 或区域特征，维度和语义分布与文本 embedding 不同。projector 用简单映射把视觉特征投到 LLM 空间；Q-Former 用可学习 query 从视觉特征中抽取少量语义 token；cross-attention 则让文本生成过程中动态读取视觉信息。核心目标是控制 token 成本并让 LLM 能稳定利用视觉证据。

</details>

## 5. 近 7 天新技术：刚刚，全球首个“事件级预测”具身智能世界模型来了！

- 题目：量子位 2026-05-29 报道了 WALL-WM 这类“事件级预测”具身智能世界模型。面试中如果让你解释它，为什么“按事件建模”可能比“按固定帧预测”更适合机器人？
- 考察点：具身智能、世界模型、事件级建模、多模态动作生成

<details>
<summary>参考答案</summary>

固定帧预测把动作切成均匀时间片，容易让模型学习局部像素变化，而不是任务中的关键物理阶段。事件级建模把抓取、接触、抬升、放置等语义边界作为预测单位，更贴近机器人真实执行中的状态转移。它的潜在优势是减少冗余预测、保留高层意图、改善长尾动作学习，并让上层规划和底层控制之间的接口更清晰；难点则在事件标注、实时闭环、跨本体泛化和安全验证。

</details>

## 6. 编程题（ML）：实现多分类交叉熵损失

- 链接：[Deep-ML 134: Multi-Class Cross-Entropy Loss](https://www.deep-ml.com/problems/134)
- 面试场景：深度学习基础面试中常见，用来考察数值稳定性和向量化。
- 函数签名：`softmax_cross_entropy(logits, labels)`
- 题目：给定 logits，形状为 (n, c)，以及每个样本的类别下标 labels，返回平均交叉熵损失和 logits 梯度。
- 样例：
  - `logits=[[2,1,0]], labels=[0] -> loss≈0.4076, grad≈[[-0.3348,0.2447,0.0900]]`
- 约束：使用 NumPy；需要减去每行最大值保证稳定；参考解法 30 行以内。

<details>
<summary>参考答案</summary>

```python
import numpy as np


def softmax_cross_entropy(logits, labels):
    logits = np.asarray(logits, dtype=float)
    labels = np.asarray(labels, dtype=int)
    shifted = logits - logits.max(axis=1, keepdims=True)
    exp = np.exp(shifted)
    probs = exp / exp.sum(axis=1, keepdims=True)
    n = logits.shape[0]
    loss = -np.log(probs[np.arange(n), labels] + 1e-12).mean()
    grad = probs.copy()
    grad[np.arange(n), labels] -= 1
    grad /= n
    return loss, grad
```

</details>
